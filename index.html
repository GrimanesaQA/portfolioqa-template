Projects

Web Application – Manual and Automated Testing

I participated in the testing of a web application as part of a collaborative project during a QA Bootcamp. The work was organized using Agile methodologies, with Jira used to manage tasks and track progress during the different sprints.

I designed and executed manual test cases based on user stories and acceptance criteria. I also helped identify and report bugs, documenting them in Jira so they could be reviewed and fixed by the development team.

Gherkin scenarios were used to describe the expected behavior of the application in a clear and structured way.

I also contributed to the test documentation in Confluence, including information about the test plan, test results, and the quality status of the product. At the end of each sprint, I participated in preparing the release in Jira and documenting the sprint closure.

QA Portfolio – Web Development and Software Testing

As part of my training in a QA Bootcamp, I developed this QA portfolio website to present my projects and learning in software testing.

The website was created using HTML and edited in Visual Studio Code. The project is managed using GitHub and GitHub Desktop for version control.

The portfolio is published using GitHub Pages, allowing the website to be accessible online and demonstrating the workflow from development to deployment.

This project showcases my progress in learning manual testing, test documentation, and working in collaborative environments using Agile methodologies.
